Chainage User Guide

Everything you need to plan, analyse, and communicate linear infrastructure programmes — from your first activity to Monte Carlo risk analysis.

💡
New to Chainage? Start with the Quick start guide and have a working programme in under 5 minutes.

What is a time-chainage chart?

A time-chainage chart (also called a time-distance diagram or linear schedule) plots work activities against both time (vertical axis) and chainage — location along a linear route (horizontal axis).

Unlike a Gantt chart which only shows when something happens, a time-chainage chart shows where it happens too. This is essential for roads, rail, tunnels, pipelines, and any project where production rates and location conflicts matter.

A typical time-chainage chart — activities show both when and where work happens. Steeper slopes = faster production rates.

The slope of an activity line represents its production rate. A steeper line means work is progressing faster along the chainage. Parallel lines mean activities are running at the same rate — useful for showing chase sequences where one trade follows another.

Quick start guide

Get a working programme in 5 minutes:

  1. Set your timeline — go to Setup → Settings and enter your project start and end dates, chainage range, and spacing.
  2. Add your first activity — go to Activities → Editor and fill in the form: name, start/end dates, start/end chainage, shape, and colour. Click Save Activity.
  3. Add more activities — repeat, or switch to Grid view to add many activities at once by typing or pasting from Excel.
  4. Link activities — go to Logic → Relationships to create Finish-to-Start or other dependency links between activities.
  5. Run the scheduler — click the Schedule button in the toolbar. Activities with dependencies will be pushed to respect their constraints.
  6. Export — go to Output → Dashboard to export as PNG, PDF, or Excel.
✓
Load the sample project from the Open menu to see a fully built programme with zones, dependencies, and risks — great for exploring features.

The interface

Chainage interface — sidebar navigation (left), form editor (top), live canvas chart (bottom)

The interface has three main areas:

  • Sidebar — navigate between all sections of the app. Grouped into Setup, Programme, Activities, Logic, Analyse, and Output. Click ☰ to collapse it.
  • Main panel — the active section (editor, analysis, settings etc.) appears here above the canvas.
  • Canvas — the live chart. Always visible. Updates in real time as you make changes.

Project settings

Before adding activities, configure your project in Setup → Settings:

Min / Max ChainageThe start and end of your route in metres (or any consistent unit). E.g. 0 to 10000 for a 10km road.

Reverse ChainageFlip the chainage axis so it runs right-to-left. Useful for some UK road conventions.

⚠
Settings changes only apply after clicking Apply Settings. Both Timeline Start and End must be set before the button activates.

Adding activities

Go to Activities → Editor. The editor has two modes — switch between them with the Form and Grid buttons:

Form mode

Add or edit one activity at a time. Fields:

  • Name — displayed on the chart if labels are enabled
  • Start / End date — the time axis position
  • Start Ch / End Ch — chainage positions (route start and end of the activity)
  • Shape — how the activity is drawn (see below)
  • Colour — hex colour for the activity bar or line
  • Progress % — shown as a dark overlay on the bar
  • Label — show activity name: no / left / centre / right
  • Groups — assign to categories and subgroups for filtering and colouring
  • Calendar — working day calendar for scheduling
  • Resources — assign named resources from your Resource Library

Editing on the canvas

You can also interact with activities directly on the canvas:

  • Click an activity to select it and load it into the form editor
  • Drag to move it in time and chainage simultaneously
  • Drag the top/bottom edge to change start/end date
  • Drag the left/right edge to change start/end chainage
  • Right-click for a context menu with edit, delete, and linking options
  • Shift+click multiple activities to multi-select

Categories & groups

Groups let you organise activities into Categories (top level) and Sub-groups (nested under a category). Groups can be colour-coded and toggled for visibility on the chart.

Example structure:

Zone A (category)
  ├── Earthworks (subgroup)
  ├── Drainage (subgroup)
  └── Paving (subgroup)
Zone B (category)
  ├── Drainage (subgroup)
  └── Structures (subgroup)

Assign activities to groups via the form editor's Groups picker, or directly in the grid view's Groups column using the format Category / Subgroup. Separate multiple with semicolons: Zone A / Drainage; Zone B / Paving.

Grid view (bulk editing)

Switch to Grid in the editor to edit all activities in a spreadsheet-like table. You can:

  • Paste from Excel — copy rows in Excel and press Ctrl+V in the grid
  • Edit the Groups column — type Zone A / Drainage to assign and auto-create groups
  • Right-click a column header to sort, filter, or hide columns
  • Drag the fill handle (bottom-right corner of a cell) to copy values down
  • Click + 5 Rows to add blank rows for new activities
  • Click Clear Empty Rows to remove blank entries
💡
Date format in the grid is DD/MM/YYYY. The grid will highlight invalid dates in red.

Dependency links

Go to Logic → Relationships to create links between activities. Chainage supports four standard CPM dependency types:

TypeMeaningExample
FS — Finish to StartB cannot start until A finishesDrainage starts after Earthworks finishes
SS — Start to StartB cannot start until A startsKerbing starts with Drainage
FF — Finish to FinishB cannot finish until A finishesLandscaping finishes with Paving
SF — Start to FinishB cannot finish until A startsRare — used for just-in-time constraints

You can add a lag (positive or negative) to any link. A lag of +5 on an FS link means B starts 5 days after A finishes.

Creating links quickly

On the canvas, right-click an activity and select Link selected in order (FS) after shift-clicking multiple activities. This chains them in the order you clicked them.

The scheduler

Click ▶ Schedule in the toolbar to run the forward-pass scheduler. It pushes activity start dates forward to satisfy all dependency constraints.

The scheduler pushes activities forward to satisfy FS, SS, FF, and SF constraints
⚠
The scheduler moves activities forward in time only. It will not move activities earlier than their manually set start date. Run it after adding or changing dependency links.

Calendars

Calendars define which days are working days for an activity. Go to Programme → Calendars to create and manage calendars.

Two built-in calendars:

  • 7-Day Standard — all days are working (default)
  • 5-Day Standard — Mon–Fri only

Create custom calendars and click individual days to mark them as non-working (bank holidays, site shutdowns etc.).

When you assign a non-7-day calendar to an activity and run the scheduler, Chainage stretches the activity's end date to preserve the intended number of calendar days. Switch back to 7-day and the end date reverts exactly.

Critical path

Enable Crit Path in the toolbar toggles to highlight activities with zero or negative float in red. The critical path is computed using the CPM backward pass — consistent with the Float Analysis tab.

Resources & cost

Chainage has two resource modes:

  • Simple mode — enter Workers/day, Labour rate, Equipment hire/day, and Material cost directly on each activity in the form editor or grid.
  • Library mode — build a named resource library (👷 Labour, 🚜 Plant, 🧱 Materials) and assign specific resources to activities. Rates and quantities are tracked per resource.

Costs are aggregated into the Cost and Combined chart views as monthly bars and a cumulative S-curve.

Zone occupancy

Zones define physical areas of your route — e.g. Zone A: Ch 0–200, Zone B: Ch 200–500. Go to Programme → Zones to create them.

The Zone Occupancy view shows a Gantt-style chart of which activities are active in each zone over time. Red dashed outlines indicate activities that overlap in the same zone at the same time — a potential clash requiring coordination.

Monte Carlo risk analysis

Chainage includes a built-in Monte Carlo simulation engine using the PERT Beta distribution — the industry-standard approach for schedule risk analysis.

Monte Carlo S-curve — 10,000 simulations produce P50/P80/P90 completion dates. Hover to read off any probability.

To use Monte Carlo:

  1. Go to Analyse → Risk Register and add risks — set Likelihood, Impact, and which activities are affected.
  2. Switch to the Monte Carlo view in the toolbar.
  3. The simulation runs automatically and shows the S-curve with P50, P80, and P90 projected completion dates.
  4. Hover over the curve to read off the probability of completing by any given date.

Baselines & revisions

In Analyse → Baselines, you can snapshot the current programme as a baseline. The baseline appears as faint ghost bars behind the live activities, making progress and slippage immediately visible.

In Analyse → Revisions, create a named revision of the programme (e.g. "Rev 3 — April submission") to track how the programme has changed over time.

Date markers

Date markers are horizontal lines drawn across the full chainage width at a specific date — useful for contract milestones, key handovers, or access restrictions. Go to Programme → Date Markers to add them. Each marker has a name, date, and colour, and shows as a labelled pill on the canvas.

P6 / Primavera sync

Chainage can sync activity dates from Primavera P6 using P6 Activity IDs:

  1. Enter a P6 ID on each activity in the form editor or grid view.
  2. Export updated dates from P6 as a CSV or copy them from P6's columns.
  3. Go to Output → Sync & Import and paste or upload the P6 data.
  4. Chainage matches activities by P6 ID and updates their start/end dates.
💡
Full P6 XML import is on the roadmap. Let us know if this is a priority for your team.

Pop-out live preview

Click the ⧉ button in the header to open a live preview of the chart in a separate browser window. This is useful when working on a second monitor — the popout updates automatically every time you make a change. All 7 chart views are available in the popout.
