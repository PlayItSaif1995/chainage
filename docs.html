Chainage User Guide

Everything you need to build, analyse, and communicate linear infrastructure programmes — from your first activity to Monte Carlo risk analysis.

💡
New to Chainage? Start with the Quick start guide and have a working programme in under 5 minutes. Or press the 🚀 Tour button in the app for a guided walkthrough.

What is a time-chainage chart?

A time-chainage chart (also called a time-distance diagram or linear schedule) plots work activities against two axes simultaneously:

  • Vertical axis — Time: when work happens (dates, months)
  • Horizontal axis — Chainage: where it happens (location along the route, in metres)

Unlike a Gantt chart which only shows when something happens, a time-chainage chart shows where it happens too. For roads, rail, tunnels, and pipelines, this spatial dimension is essential — it lets you see production rates, zone clashes, and sequencing conflicts that a Gantt simply cannot reveal.

A typical time-chainage chart — steeper slopes = faster production rates. Activities show both when and where work happens.

The slope of a line = its production rate. Steeper = faster progress along the chainage. Parallel lines mean activities are running at the same rate — useful for chase sequences.

Quick start guide

Get a working programme in 5 minutes:

  1. 1
    Set your timeline — go to Setup → Settings, enter project start/end dates, chainage range (e.g. 0–10000), and spacing. Click Apply Settings.
  2. 2
    Add an activity — go to Activities → Editor, fill in name, dates, chainages, shape, and colour. Click Save Activity to see it appear on the canvas instantly.
  3. 3
    Add more activities — switch to Grid view to bulk-add by typing or pasting from Excel. Or press D to draw activities directly on the canvas.
  4. 4
    Create groups — go to Activities → Grouping to assign activities to colour-coded categories and sub-groups.
  5. 5
    Add dependencies — go to Logic → Relationships to link activities with FS/SS/FF/SF constraints.
  6. 6
    Schedule — click the green ⚡ Schedule button. The CPM engine adjusts dates to satisfy all constraints.
  7. 7
    Export — go to Output → Dashboard or click 📤 Export to download PNG, PDF, or Excel.
✓
Load a sample project from the Open menu to explore a fully built multi-zone programme with dependencies, risks, and zones pre-loaded.

The interface

Chainage has three main areas:

The Chainage interface — ① Sidebar navigation, ② View toolbar, ③ Active editing panel, ④ Live chart canvas
  • ① Sidebar — navigate all sections grouped by workflow stage. Click ☰ to collapse it for more canvas space.
  • ② Toolbar — switch between the 7 chart views. The green ⚡ Schedule button runs the CPM scheduler.
  • ③ Active panel — the current section (editor, analysis, settings) appears above the canvas.
  • ④ Canvas — the live chart. Always visible, updates in real time as you make changes.

Editing on the canvas

You can interact with activities directly on the chart:

  • Click — select and load into the form editor
  • Drag — move in both time and chainage simultaneously
  • Drag top/bottom edge — change start or end date
  • Drag left/right edge — change start or end chainage
  • Right-click — context menu: Edit, Delete, Link, Add here
  • Shift+click multiple activities — multi-select for bulk delete or linking

Draw to create NEW

The fastest way to add activities. Press D or click the ✏ Draw button in the toolbar to enter draw mode.

Draw mode — drag diagonally across the canvas to set start/end dates AND start/end chainage simultaneously. Release to name your activity.
  1. Press D or click ✏ Draw in the toolbar — cursor turns into a crosshair
  2. Click and drag diagonally across the canvas — a dashed rectangle previews the activity position
  3. Release the mouse — a small popup appears asking for a name
  4. Type the name and press Enter — the activity is created and the editor opens so you can set shape, colour, and groups
  5. Press Escape at any time to cancel draw mode
💡
After creating, draw mode exits automatically. The activity editor opens so you can set shape, colour, calendar, and groups immediately.

Categories & groups

Groups organise activities into a two-level hierarchy: Categories (top level) and Sub-groups (under a category). Groups can be colour-coded and toggled for visibility.

Zone A (category)
  ├── Earthworks (sub-group)
  ├── Drainage  (sub-group)
  └── Paving    (sub-group)
Zone B (category)
  ├── Drainage  (sub-group)
  └── Structures (sub-group)

Assign groups via the Groups picker in the form editor, or directly in the grid view's Groups column using the format Category / Subgroup. Separate multiple with semicolons:

Zone A / Drainage; Zone B / Paving
💡
If you type a category or sub-group name that doesn't exist yet, Chainage creates it automatically. Case-insensitive — zone a matches Zone A.

Grid view UPDATED

Switch to Grid in the editor to edit all activities in a spreadsheet-style table. The grid has three column tiers:

TierColumnsDefault
CoreP6 ID, Name, Start, End, Ch Start, Ch End, Shape, Colour, GroupsAlways visible
DefaultProgress %, Label, NotesVisible, hideable
AdvancedWorkers/day, Labour rate, Equipment hire, Material cost, Opacity, Outline Style/Colour/WidthHidden — use ⊞ Columns to show

Key interactions:

  • Paste from Excel — copy rows in Excel, click a cell in the grid, press Ctrl+V
  • Groups column — type Zone A / Drainage to assign and auto-create groups. Semicolons separate multiple: Zone A / Drainage; Zone B / Paving
  • Colour cell — click any colour cell to open the palette picker with 28 infrastructure-appropriate swatches
  • Fill handle — drag the bottom-right corner of a cell to copy its value down
  • Right-click column header — sort, filter, or hide that column
  • ⊞ Columns button — show or hide any column

⚠
Date format in the grid is DD/MM/YYYY. If the end date is before or equal to the start date, the cell reverts and a warning toast appears at the bottom of the screen.

Dependency links

Go to Logic → Relationships to create links between activities. Chainage supports all four CPM dependency types:

TypeMeaningExample
FS — Finish to StartB cannot start until A finishesDrainage starts after Earthworks finishes
SS — Start to StartB cannot start until A startsKerbing starts with Drainage
FF — Finish to FinishB cannot finish until A finishesLandscaping finishes with Paving
SF — Start to FinishB cannot finish until A startsRare — just-in-time constraints

Add a lag (positive or negative days) to any link type. A +5 day lag on an FS link means B starts 5 days after A finishes. A -5 day lag means B can start 5 days before A finishes.

Quick linking on canvas

  • Right-click an activity → Add Dependency to link it to the selected activity
  • Shift+click multiple activities, then right-click → Link X selected in order (FS) to chain them in the order you clicked

The scheduler

Click ⚡ Schedule in the toolbar to run the CPM forward-pass scheduler. A confirmation dialog appears first explaining what will happen.

The scheduler pushes activities forward only — it never moves activities earlier than their user-set start date.

Key rules:

  • The scheduler only ever pushes activities forward in time — it will not move an activity earlier than its manually-set start date
  • Before each run, all successor activities are reset to their original user-set start dates — this means the schedule recalculates correctly even when a predecessor shrinks (e.g. when switching from a 5-day back to 7-day calendar)
  • Press Ctrl+Z immediately after scheduling to undo the entire operation

Calendars

Calendars define which days are working days for an activity. Go to Programme → Calendars to create and manage them.

Two built-in calendars:

  • 7-Day Standard — all days working (default)
  • 5-Day Standard — Monday to Friday only

Create custom calendars by clicking individual days on the month-grid to mark them as non-working — for bank holidays, site shutdowns, seasonal restrictions.

When you assign a non-7-day calendar to an activity and run the scheduler, Chainage stretches the activity's end date to preserve the intended number of calendar days, skipping non-working days. Switch back to 7-day and the end date reverts exactly.

💡
Each activity can have its own calendar — useful when different trades work different shift patterns on the same project.

Critical path

Enable Crit Path in the toolbar toggles to highlight activities with zero or negative float in red on the canvas. Critical path is computed using a full CPM backward pass — consistent with the Float Analysis tab.

You can also see individual activity float values by hovering on the Float heatmap view, or by opening Analyse → Float Analysis for the full programme report.

Float analysis

Float = how much an activity can slip before it delays the project end date. Chainage computes float using a full CPM backward pass across all dependency links.

Float heatmap colour scale — hover any activity in Float view to see its exact float value

Access float data two ways:

  • Float heatmap view — colour-codes the entire canvas. Hover any activity to see its float value in a tooltip.
  • Float Analysis tab (Analyse → Float Analysis) — full programme report showing total float for every activity, with scorecards identifying critical and near-critical chains.

Resources & cost

Two resource loading modes:

Simple mode

Enter directly on each activity in the form editor or grid view:

  • Workers/day — shown in the resource histogram
  • Labour rate/day — cost per worker per day (£)
  • Equipment hire/day — daily plant/equipment cost (£)
  • Material cost (total) — fixed cost for the activity (£)

Library mode

Go to Programme → Resource Library to create named resources (Labour, Plant, Material types) with specific rates. Assign them to activities with quantities. Library mode gives more granular cost tracking across the programme.

Cost and resource views

  • Resources view — daily worker histogram. Hover bars to see exact worker-day counts.
  • Cost view — monthly bars split by Labour / Equipment / Materials, with a cumulative S-curve overlaid. Hover to see the breakdown per month.
  • Combined view — resources and cost S-curve side by side.

Zone occupancy

Zones define named physical sections of your route — e.g. Zone A: Ch 0–250, Zone B: Ch 250–500. Create them at Programme → Zones.

Zone Occupancy view — red dashed outlines indicate two activities sharing the same zone at the same time (a clash)

The Zone Occupancy view shows a Gantt-style chart of which activities are active in each zone at any point in time. Red dashed outlines highlight clashes — two activities competing for the same physical space simultaneously. Resolve these at the planning stage before they cause delays on site.

Monte Carlo risk analysis

Chainage includes a built-in Monte Carlo simulation engine using the PERT Beta distribution — the same statistical model used by Primavera Risk Analysis and @Risk.

Setting up risks

  1. Go to Analyse → Risk Register
  2. Add risks — each with: title, likelihood (1–5), impact (1–5), description, affected activities, and optimistic/likely/pessimistic delay estimates in days
  3. Optionally assign risks to a correlation group — correlated risks fire together in the simulation, producing a statistically sound result
Monte Carlo S-curve with P50/P80/P90 completion date lines — hover anywhere on the curve to read off probability at any delay

Reading the results

  • P50 — 50% probability of completing by this date. The most likely outcome.
  • P80 — 80% confidence date. Industry standard for programme risk reporting.
  • P90 — 90% confidence. Often used for contractual completion with contingency.

Hover anywhere on the S-curve to read off the probability of completing by any given delay. The simulation runs 10,000 iterations using Marsaglia-Tsang gamma sampling — the same underlying mathematics as dedicated risk tools, built directly into your programme.

Baselines & revisions

Baselines

Go to Analyse → Baselines to snapshot the current programme. The baseline appears as faint ghost bars behind live activities, making progress and slippage immediately visible on the canvas.

Revisions

Go to Analyse → Revisions to create a named version of the programme (e.g. "Rev 3 — April submission"). Revisions track how the programme has changed over time and provide an auditable record for contractual purposes.

Date markers

Date markers are horizontal lines drawn across the full chainage width at a specific date — useful for contract milestones, access restrictions, or key handovers. Go to Programme → Date Markers.

Each marker has a name, date, and colour, and appears as a labelled pill on the canvas and in all exported outputs.

P6 / Primavera sync

Chainage can update activity dates from Primavera P6 by matching on P6 Activity IDs:

  1. Enter a P6 ID on each activity in the form editor or grid view (P6 ID column)
  2. Export updated dates from P6 as a CSV
  3. Go to Output → Sync & Import and upload or paste the P6 data
  4. Chainage matches activities by P6 ID and updates their start/end dates
💡
Full P6 XML import is on the roadmap. Let us know if this is a priority for your team.

Pop-out live preview

Click the ⧉ button in the header to open a live preview of the chart in a separate browser window. The popout:

  • Updates automatically within 1 second of any change in the main app
  • Shows all 7 chart views — switching in the popout switches the main app view too
  • Supports hover tooltips for all views (float values, cost breakdowns, zone clashes, Monte Carlo probabilities)
  • Scales to fill the popout window — ideal for a second monitor

The ⧉ button turns blue when the popout is open. Close the popout window to reset it.
